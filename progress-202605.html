一、試做專題前端並尋找後端資源

1. 確立專案架構與開發策略

  • 前後端分離：決定將「全方位程式檢定助理」拆分為前端（畫面體驗）與後端（核心邏輯與 AI 模型），清楚定義各自職責。
  • API 優先（API-First）：確立了先開發後端 API 的策略，讓後端成為一個獨立運作的「服務總機」，未來無論接上網頁或 App 都能暢通無阻。

2. 成功在本機端喚醒 AI 大腦

  • 環境建置：在 VS Code 編輯器中，成功安裝了 transformers 與 PyTorch 等核心 AI 套件，完成開發環境配置。
  • 模型下載與推論：從 Hugging Face 下載 Qwen2.5-Coder-1.5B-Instruct（約 3GB）模型並成功在本機完成推論測試。
  • 角色扮演測試：撰寫第一支 Python 測試程式，成功讓 AI 扮演「導師」，精準且委婉地找出程式碼中缺少的「冒號」，驗證 Prompt（提示詞）策略完全有效。

二、模型測試與 API 實作

3. 架設 FastAPI 伺服器總機

  • API 封裝：將原本單純的 AI 推論邏輯，正式裝進 FastAPI 框架中，建立了可以接收外部資料的 POST /api/diagnose 端點，完成 API 接口實作。
  • 伺服器啟動：學會使用 uvicorn 指令，將程式變成 24 小時待命的伺服器（運行於 localhost:8000）。
  • 內部連線測試：透過 FastAPI 內建的 Swagger UI（/docs），成功發送 JSON 測試包，並完美收到 200 OK 與 AI 導師的回覆。

4. 模型測試與功能優化

  • 模型探索：從 Hugging Face Space 尋找合適的模型範例，並完成相關環境模組安裝。
  • 功能試跑：成功跑通他人分享的 Qwen 推論範例，並測試模型對程式碼診斷的效果。
  • 後處理優化：針對試跑結果撰寫文字過濾功能，確保 AI 回覆內容簡潔正確。

三、內網穿透，API 正式上線全球

  • Ngrok 部署：下載並成功設定 Ngrok 礦工程式，綁定了專屬通行證，完成雲端穿透準備。
  • 打通地道：成功執行指令，將電腦裡的 8000 埠對外開放，獲得一串專屬的 https://...ngrok-free.dev 公開網址，後端正式具備了商業上線的連線能力。

四、製作 ICFI 報告簡報

  • 依據專案進度與成果，整理並製作 ICFI 相關報告簡報，呈現本月開發成果與技術架構說明。
本月總結：「全方位程式檢定助理」現已擁有一個實體存在、隨時待命，且全世界都能連線呼叫的 AI 大腦。完成了從本機模型部署、FastAPI 封裝、到雲端公開上線的完整後端架構建置，為後續前端串接與功能擴展奠定了堅實基礎。

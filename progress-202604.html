1. 基礎雛型架構與 Claude API 串接測試

我們成功完成系統基礎架構開發，並透過 Windows PowerShell 環境順利串接 Claude API 。此階段重點在於驗證 AI 代理人的溝通效能，為後續高職生程式學習平台的互動核心奠定穩定基礎 。

2. 專題正式帳號建置與管理

為了確保開發流程的專業性與協作效率，我們統一建立了專題專用的 Google 與 GitHub 帳號。這有助於集中管理開發資源、API 金鑰及各項雲端工具，確保專案資產的完整與安全性 。

3. GitHub 專題展示網站開發

我們在 GitHub 上建立了專屬的進度記錄與展示網站，並初步完成整體介面實作 。該網站不僅是專題成果的對外窗口，也紀錄了開發過程中的技術選型與系統設計邏輯 。

4. 進度頁面動態維護與 HTML 優化

針對展示網站，我們在 HTML 中規劃了清晰的月份劃分，並利用 Claude 輔助修改程式碼，以實現快速更新 。透過與 AI 的協作，我們能更有效率地調整網頁佈局，確保每月的研發進度能即時且準確地呈現 。

5. TWCC ONE AI 平台模型試跑

我們導入了 TWCC ONE AI 平台進行初步的模型訓練與試跑測試 。利用其一站式的資料準備、模型建立與部署特色，我們得以在高效能環境下驗證 AI 模型在程式教學應用上的落地可行性 。
